# Azure Security Lab – Identity & Threat Protection

## 🔐 Overview
This project demonstrates hands-on experience with Microsoft Entra ID (Azure Active Directory) and Microsoft Defender for Cloud, focusing on identity security, access control, and cloud threat protection.

The lab simulates a real-world enterprise environment by implementing core security controls such as Multi-Factor Authentication (MFA), Conditional Access policies, and Self-Service Password Reset (SSPR).

---

## 🎯 Objectives
- Create and manage users in Microsoft Entra ID
- Organize users using security groups
- Implement Multi-Factor Authentication (MFA)
- Configure Conditional Access policies (Zero Trust)
- Enable Microsoft Defender for Cloud
- Review and respond to security recommendations
- Enable Self-Service Password Reset (SSPR)

---

## 🏗️ Environment
- Microsoft Azure (Free Tier Subscription)
- Microsoft Entra ID
- Microsoft Defender for Cloud
- Windows Server Virtual Machine (Azure)

---

## ⚙️ Key Configurations

### 👤 Identity Management
- Created cloud users within Microsoft Entra ID
- Built security groups to simulate department-based access
- Added users to groups for role-based access control (RBAC)

---

### 🔐 Security Controls (Zero Trust)
- Enforced Multi-Factor Authentication (MFA) using Conditional Access
- Created policy targeting all users and all cloud applications
- Strengthened login security and reduced attack surface

---

### 🛡️ Threat Protection
- Enabled Microsoft Defender for Cloud
- Monitored security posture and recommendations
- Reviewed alerts and hardening suggestions for Azure resources

---

### 🔄 User Self-Service
- Enabled Self-Service Password Reset (SSPR)
- Configured authentication methods (email, phone)
- Reduced dependency on IT support for password resets

## 🧠 Key Takeaways
- Identity is the first line of defense in cloud security
- Conditional Access is critical for enforcing Zero Trust
- MFA significantly reduces risk of unauthorized access
- Cloud-native tools provide continuous security insights
- Automating user self-service improves operational efficiency
